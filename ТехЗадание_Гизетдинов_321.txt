Техническое задание
ТЕМА: “Курорты Краснодарского края”
1.	ЦЕЛЬ ПРОЕКТА.
Разработка приложения для просмотра курортов и покупки путевок в Краснодарском крае.
2.	ИДЕЯ ПРОЕКТА.
Компания хочет продавать путевки на курорты Краснодарского края. Компания выходит на новый уровень и заказывает приложение для удобства пользователей. Компании нужно, чтобы в приложении были все данные о ней и основной функционал продажи путевок, трансфера. Компания требует вход/регистрацию в приложении для того, чтобы просматривать историю покупок путевок.
3.	ЗАДАЧА ПРОЕКТА.
Главная задача этого проекта – это просмотр путевок и онлайн бронирование номеров в отеле. Также в этом проекте присутствуют задачи:
- Просмотр акций на данный момент;
- Просмотр контактов;
- Организация трансфера.
Проект представляет собой клиентское приложение с встроенной базой данных.
4. РАЗРАБОТКА
В этом приложение будут реализованы такие возможности, как:
- Просмотр доступных курортов;
- Покупка путевки; 
-  Вход/Регистрация.
В приложении используются средства:
- Система для построения клиентсикх приложений "WPF";
- Система управления реляционными базами данных “Mongo DB”.
Главная страница(Клиент):
1. Информация о курорте(поля):
1.1 Название;
1.2 Дата;
1.3 Место. 
2. Кнопка регистрации/входа;
3. Кнопка покупки путевки.;
	4. Кнопка навигации.
Навигационная панель(Клиент):
1. Кнопка главная страница; 
2. Кнопка перехода к странице заказанной путевки;
3. Кнопка перехода к странице акций (для просмотра текущих акций);
4. Кнопка перехода к странице инфо (прочитать инфо о компании);
5. Кнопка перехода к странице личный кабинет.
Страница Путевки:
	1. Инфо о заказанной путевке(поля):
1.1 Название;
1.2 Код путевки;
1.3 Дата вылета.
	2. Кнопка отмены заказа.
Страница Акций: 
	1. Инфо о доступных акциях(поля):
1.1	Название;
1.2	Скидка;
1.3	Дата действия акции.
Страница Инфо:
	1. Инфо о компании(поля);
		1.1 Название;
		1.2 Описание;
	2. Лого.
Страница Личный Кабинет:
 	 1. Инфо о аккаунте(поля):
		1.1 ФИО;
		1.2 Логин;
		1.3 Email;
		1.4 Мобильный телефон.
	2. Кнопка выхода.
Форма покупки путевки:
1. Текстовые поля:
	1.1 Ввод банковской карты. 
2. Кнопка очистки;
3. Кнопка подтверждения.
Приложение “WPF”.
Общие:
1. Цветовая палитра бело-синий.
2. Регистрация/редактирование клиентов.
3. Редактирование/отмена записей.
